# Commit Template - Short Format

Use for simple, focused changes.

**Plaintext only**: No markdown formatting. Use dashes and ALL CAPS headers.

```plaintext
type(scope): brief description in imperative mood

CHANGES
- [Key change 1]
- [Key change 2]
```

## When to Use

- Single-purpose changes
- No breaking changes
- No issues to close
- Self-explanatory modifications
